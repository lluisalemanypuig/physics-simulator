Detailed Description

Abstract spring mesh.

This represents a set of particles::mesh_particle where one particle is attached to some other particles of the same mesh with a spring. The movement of one particle in the spring, then, affects the other particles it is attached to. Each particle has a local index i that ranges between 0 and the number of particles - 1.

All the springs of the mesh are described similarly with the following coefficients:

  • elasticity parameter (see Ke).
  • damping factor (see Kd).

Some coefficients are common to all the particles:

  • bouncing coefficient (see bouncing).
  • friction coefficient (see friction).

There are internal forces that can be simulated. These are dependent on each type of mesh.

void physim::meshes::mesh::allocate ( size_t  n,
float  Kg = 1.0f 
)

Allocates memory for n particles.

The previous contents of this spring are cleared (see clear).

Every particle is initialised with its default constructor. After that, they are assigned an index within the mesh.

This index is local, that is, the first particle of every 1-dimensional mesh has index 0, the second particle has index 1, ...

The mass of each particle can also be initialised at this point. The mass of the mesh (param Kg) is divided uniformly among the n particles.

◆ make_initial_state()

virtual void physim::meshes::mesh::make_initial_state ( )
pure virtual

Builds the initial state of the mesh.

After assigning each of this mesh's particles a position, the initial state needed for proper simulation is computed.

This may include the initial distances between the particles at each end of every simulated spring.

Each type of mesh has to implement this method for a proper simulation of that type of mesh.

Precondition
In general, all this mesh's particles must have been initialised (say, given an initial current position).

Implemented in physim::meshes::mesh2d_regular, and physim::meshes::mesh1d.

◆ update_forces()

virtual void physim::meshes::mesh::update_forces ( )
pure virtual

Update the forces generated within the mesh.

This method updates the forces acting on all particles. This update depends on the neighbouring information determined by the type of mesh.

This method does not update positions or velocities.

Precondition
The modification of a particles' force may assume that all particles start with null force (force equal to 0 in the three axes).
This method is called before computing the forces acting on the particles due to force fields.

Implemented in physim::meshes::mesh2d_regular, and physim::meshes::mesh1d.
