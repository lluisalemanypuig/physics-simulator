Detailed Description

Class implementing an agent particle.

An agent particle is useful for defining, using a physically-based model, the movement of an agent (say, a human(oid), car, ...) through an environment, in the presence of other agents and obstacles.

Although it is a type of sized particle (see sized_particle), which in turn is a type of free particle (see free_particle), its starting time (see free_particle::starttime) is ignored. Its lifetime, however, it is not (see free_particle::lifetime). The simulation of an agent particle depend on its behaviour (see agent_particle::behaviour).

void physim::particles::agent_particle::apply_behaviours ( math::vec3 &  weighted_steering) const

Computes a weighted steering vector force.

This vector is the weighted average over the different steering vectors obtained by the different behaviours activated.

The behaviours considered in this function are:

  • agent_behaviour_type::seek (function seek_behaviour(math::vec3&)const ).
  • agent_behaviour_type::flee (function flee_behaviour(math::vec3&)const ).
  • agent_behaviour_type::arrival (function arrival_behaviour(math::vec3&)const ).

The resulting vector of each behaviour will be accumulated to the output vector of this function.

Recall that the target is stored in target.

Parameters
[out]weighted_steeringWeighted steering vector.

void physim::particles::agent_particle::unaligned_collision_avoidance_behaviour ( const std::vector< agent_particle > &  agents,
math::vec3 &  v 
) const

Computes the collision avoidance steering force.

This function must compute a velocity vector multiplied by weight ucoll_weight. The result must be assigned to v.

The vector computed is meant to provide the agent a means of avoiding collision with other agents 'locally', that is, the ability of choosing what to do without the help of an oracle.

Only some agents are considered. Those ignored are listed below. FOV stands for Field of View, it is aligned at this agent's velocity vector and is of 90 to its left and 90 to its right.

The following agents are ignored:

  • agents too far away (the distance between the agents is more than ucoll_distance, where the distance considered is the distance between the current positions minus the sum of both agent's radius)
  • agents whose predicted position is outside the FOV.
  • agents whose current position is outside the FOV.
Parameters
[in]agentsAll the agents in the simulation.
[out]vUnaligned collision avoidance steering vector.
Precondition
Vector v may not be initialised to 0.
