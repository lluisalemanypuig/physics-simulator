10 /* The readability of the code resulting from using these
11  * macros is not too good. However, each of them has a clear
12  * 'syntax', so to speak.
13  *
14  * First, start with '__pm', which makes reference to the
15  * namespaces physim::math. Then follows the type of operation:
16  * 'add', 'sub', 'mul', 'div'. That is, '__pm3_add', '__pm3_sub',
17  * '__pm3_mul', '__pm3_div'. Multiplication and division between
18  * two vectors are always component-wise.
19  *
20  * After the operation type comes the parameter specification.
21  * There are two basic types of parameters, though three letters
22  * are used:
23  * - 'v' to indicate vector (see @ref vec3),
24  * - 's' to indicate scalar, and
25  * - 'c' to indicate 3 scalars.
26  *
27  * These letters are combined to specify the input parameters
28  * and the type of operation between them. Notice that there
29  * is always one parameter that is omitted: the output vector.
30  * If they are separated by a '_' (e.g. 'v_v') they are related
31  * with the operation specified. If they are together (e.g. 'vv')
32  * they are both multiplied.
33  *
34  * For example, the macro:
35  * * __pm3_mul_v_v: indicates the component-wise multiplication
36  * of two vectors.
37  * * __pm3_add_v_vs: indicates the component-wise addition of a
38  * vector ('v') and the result of multiplying a vector by a
39  * scalar ('vs').
40  *
41  * The first vector of these operations always has a leading '+'.
42  *
43  * Some operations of the form 'u <- u _op_ v' are defined in a
44  * number of macros. These were added an 'acc' between the
45  * indicator of the operation and the parameters.
46  * Therefore, the macros:
47  * * __pm3_sub_c: implements 'u <- u - (x,y,z)' which is equivalent
48  * to 'u -= (x,y,z)'.
49  * * __pm3_mul_s: implements 'u <- u*s' which is equivalent to
50  * 'u *= s', for a scalar value 's'.
51  *
52  * For longer operations, the operator indicated after 'pm'
53  * only refers to the "sign of the accumulation", that is:
54  * - add for += X
55  * - sub for -= X
56  * - mul for *= X
57  * - div for /= X
58  * The operator between the vectors in the expression X is
59  * always the sum (+).
60  *
61  * Below follow the details on the defintion of each macro.
62  */
